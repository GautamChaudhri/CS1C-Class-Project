\doxysubsection{Detailed Description}
Implements the Crow web service for handling shapes, render area, user, and testimonial data. 

This web service provides the following endpoints, grouped by entity\+:
\begin{DoxyItemize}
\item TEST GET "{}/"{} \+: A test endpoint that returns a greeting.
\item Shapes Endpoints\+:
\item GET /shapes \+: Returns the JSON-\/formatted shapes data from ../../database/shapes.json.
\item POST /shapes \+: Accepts JSON data to update the shapes file (../../database/shapes.json).
\item DELETE /shapes-\/all \+: Clears all shapes data by truncating the shapes.\+json file.
\item Render Area Endpoints\+:
\item GET /render\+\_\+area \+: Returns the JSON-\/formatted render area data from ../../database/render\+\_\+area.json.
\item POST /render\+\_\+area \+: Accepts JSON data to update the render area file (../../database/render\+\_\+area.json).
\item DELETE /render\+\_\+area-\/all \+: Clears all render area data by truncating the render\+\_\+area.\+json file.
\item Users Endpoints\+:
\item GET /users \+: Returns the JSON-\/formatted user data from ../../database/users.json.
\item POST /users \+: Accepts JSON data to update the user file (../../database/users.json).
\item DELETE /users-\/all \+: Clears all user data by truncating the users.\+json file.
\item Testimonials Endpoints\+:
\item GET /testimonials \+: Returns JSON-\/formatted testimonial data from ../../database/testimonials.json.
\item POST /testimonials \+: Accepts JSON data to update the testimonials file (../../database/testimonials.json).
\item DELETE /testimonials \+: Clears all testimonial data by truncating the testimonials.\+json file. (Note\+: Original was DELETE /testimonials, implying delete all)
\end{DoxyItemize}

\begin{DoxyNote}{Note}
Uses the Crow framework for handling HTTP requests. Make sure the database directory exists or is created.
\end{DoxyNote}
